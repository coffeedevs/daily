The biggest shift in the last day is infrastructure, not a new flagship model. OpenAI tightened local-agent ergonomics in Codex, then shipped same-day Agents SDK releases that standardize on `gpt-realtime-1.5` and expand MCP server/session support. For teams already running coding agents in terminals or workers, today’s upgrade is about fewer restarts, fewer custom wrappers, and cleaner approval config.

- **Codex reduces terminal friction in multi-turn sessions**: Mar 23 adds `Shift+Left` editing for queued tmux messages and snapshot-fork startup context injection. Use queued-turn correction and snapshot forks for pane handoffs instead of restarting sessions.
- **Codex is forcing approval-config cleanup before wider unattended use**: Mar 23 removes the `smart_approvals` alias migration, following fresh exec-policy test coverage on Mar 22. Rename legacy keys now and pin wrapper policies explicitly before your next Codex upgrade.
- **OpenAI Agents Python 0.13.0 resets the default realtime floor**: New websocket agents now default to `gpt-realtime-1.5`, add MCP resource reads plus resumable streamable HTTP sessions, and fix run-loop races. Drop custom MCP shims that only expose resources or session resume, then retest long-lived workers.
- **OpenAI Agents JS 0.8.0 brings the same baseline to Node stacks**: JS agents now default to `gpt-realtime-1.5`, add MCP resource wrappers and streamable session IDs, and recover segmented assistant output more reliably. Upgrade Node runtimes that depend on MCP or handoffs, then recheck finalization and filter logic.
