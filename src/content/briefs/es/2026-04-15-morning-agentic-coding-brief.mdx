Los cambios útiles de hoy apuntan a que los agentes sean más confiables para correr, reanudar y depurar. El patrón se repite: más comportamiento en código, más estado que sobrevive interrupciones y menos dependencia de un solo prompt gigante.

- **Claude Code vuelve más barata la sesión interrumpida**: `2.1.108` suma `/recap`, lleva el prompt cache TTL a 1 hora en API key, Bedrock, Vertex y Foundry, y expone slash commands built-in vía Skill tool. Activá la caché larga y tratá esos comandos como primitivas reutilizables de agente.
- **Claude Code sigue bajando la fricción de reanudar**: `2.1.109` mejora el indicador de extended thinking, y `2.1.108` acota `/resume` al directorio actual y baja el uso de memoria al leer archivos. Si tu equipo deja agentes corriendo en paralelo, estandarizá workflows centrados en reanudar.
- **Codex se vuelve más amistoso para extensiones locales**: OpenAI agregó local marketplace sources, hizo filesystem-aware la carga de skills y mejoró defaults de tool search en `main`. Mové plugins y skills privados más cerca del repo y probalos como assets portables del workspace.
- **Codex agregó wire tracing opt-in para depurar realtime**: un nuevo trace target registra eventos raw de WebSocket y requests SDP de WebRTC sin ensuciar el resto de los logs. Sumá esa flag al playbook de repro para sesiones colgadas de voz, browser o background agents.
- **Google empuja arquitectura de agentes por encima de decoración de prompts**: su guía más reciente del Agent Bake-Off recomienda agentes descompuestos, código determinístico alrededor del modelo, contratos de herramientas explícitos y loops rápidos de evaluación. Refactorizá agentes de “hacé todo” en etapas planner/executor/evaluator antes de sumar más prompt.
