Claude Code publicó el cambio más útil para el flujo diario: los agentes ahora pueden pausar y pedir input estructurado vía Model Context Protocol (MCP) en vez de forzar todo por prompt libre. El patrón más claro es una convergencia hacia loops agénticos más controlables, mejor ergonomía en monorepos y auth más seguro alrededor de las herramientas.

- **Claude Code suma input MCP estructurado y control para monorepos**: `v2.1.76` permite que servidores MCP pidan input estructurado a mitad de tarea, agrega `worktree.sparsePaths` y expone `/effort`. Conviene mover herramientas internas de prompts largos a entradas guiadas y ajustar sparse checkouts para agentes en repos grandes.
- **Gemini CLI profundiza los flujos agente a agente**: el nightly más reciente agrega OAuth2 Authorization Code para autenticación agente a agente (A2A), activación de skills por slash commands, uso de tokens por modelo en `stream-json` y un scheduler de herramientas orientado a eventos. Actualizá tus harnesses y empezá a medir costo por modelo, no solo gasto agregado.
- **OpenAI Node agrega custom voices**: `openai-node v6.29.0` expone voces personalizadas en el cliente API. Si tu copiloto habla, podés mover la identidad de voz al producto sin mantener una capa separada de texto a voz.
- **AI SDK endurece OAuth en MCP**: `@ai-sdk/mcp@1.0.28` valida el `state` de OAuth durante la autenticación MCP, y el patch train también empujó updates del gateway. Revalidá los flujos web de sign-in para MCP y eliminá cualquier manejo laxo de `state` que haya quedado del desarrollo local.
