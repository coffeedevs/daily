El cambio más importante del último día es de infraestructura, no de un modelo estrella. OpenAI afinó la ergonomía de agentes locales en Codex y después publicó releases el mismo día para ambos Agents SDK, con `gpt-realtime-1.5` como baseline y mejor soporte de servidores y sesiones MCP. Para equipos que ya corren agentes de código en terminales o workers, el upgrade útil de hoy es menos reinicios, menos wrappers caseros y una config de approvals más limpia.

- **Codex reduce la fricción de terminal en sesiones de varios turnos**: el 23 de marzo agregó edición con `Shift+Left` para mensajes en cola dentro de tmux y startup context injection para snapshot forks. Usá corrección de turnos en cola y snapshot forks para pasar trabajo entre panes sin reiniciar sesiones.
- **Codex está forzando una limpieza de config de approvals antes de ampliar uso desatendido**: el 23 de marzo eliminó la migración del alias `smart_approvals`, después de sumar cobertura nueva para exec policy el 22 de marzo. Renombrá hoy las keys legadas y fijá policies explícitas en tus wrappers antes del próximo upgrade.
- **OpenAI Agents Python 0.13.0 redefine el baseline realtime por defecto**: los agentes por websocket ahora usan `gpt-realtime-1.5`, agregan lectura de recursos MCP y sesiones streamable HTTP reanudables, y corrigen carreras del run loop. Sacá wrappers MCP propios si sólo exponían recursos o reanudación de sesión, y reprobá workers de larga vida.
- **OpenAI Agents JS 0.8.0 lleva el mismo baseline a stacks Node**: los agentes JS ahora usan `gpt-realtime-1.5`, agregan wrappers de recursos MCP e IDs de sesión streamable, y recuperan mejor salidas segmentadas del asistente. Actualizá runtimes Node que dependan de MCP o handoffs y volvé a validar finalización y filtros.
