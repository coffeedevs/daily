Claude Code shipped the clearest workflow upgrade: agents can now pause for structured Model Context Protocol (MCP) input instead of forcing everything through free-form prompts. The broader shift is toward tighter agent control, better monorepo ergonomics, and safer auth around tool use.

- **Claude Code adds structured MCP input and monorepo controls**: `v2.1.76` lets MCP servers request structured input mid-task, adds `worktree.sparsePaths`, and exposes `/effort`. Shift internal tools from long prompts to guided input, and tune sparse checkouts plus effort defaults for large repos.
- **Gemini CLI goes deeper on agent-to-agent flows**: the latest nightly adds OAuth2 Authorization Code support for agent-to-agent (A2A) auth, slash-command skill activation, per-model token usage in `stream-json`, and an event-driven tool scheduler. Upgrade your CLI harnesses and log spend per model, not as one pooled number.
- **OpenAI Node adds custom voices**: `openai-node v6.29.0` exposes custom voices in the API client. If your coding copilot speaks, move voice identity into product code instead of maintaining a separate text-to-speech layer.
- **AI SDK tightens MCP OAuth**: `@ai-sdk/mcp@1.0.28` validates OAuth `state` during MCP auth, and the patch train also moved gateway updates forward. Retest browser-based MCP sign-in flows and remove any relaxed `state` handling left over from local development.
