The biggest shift in the past day is reliability work inside agent runtimes, not a flashy model launch. Codex is moving more app and approval plumbing into the TUI, while Gemini CLI is fixing the session and hook edge cases that usually break long-running runs. If you already use coding agents every day, the upgrade path is operational: recheck trust boundaries, approval persistence, and how you split unattended tasks.

- **Codex pulls more app tooling into the TUI**: Mar 22 commits improve app tool loading for the TUI, after Mar 21 removed more legacy app-server auth and notification handling. Expect fewer sidecar mismatches in desktop and terminal sessions; retest app-connected Codex flows and plugin bootstrap paths today.
- **Codex is redrawing trusted-run boundaries**: Mar 21 also added guardian approval context, fixed explicit plugin mentions, enabled code mode on V8, and disabled `command_might_be_dangerous` when unsandboxed. If you run Codex inside a controlled wrapper, tighten policy at the wrapper layer and re-verify approvals before broadening unsandboxed usage.
- **Gemini CLI is fixing the dead ends that stall long agent runs**: Mar 21 commits unfreeze plan mode and keep global sessions plus persistent approvals for `web_fetch`. They also allow `ask` in `BeforeTool` hooks and warn on reused browser sessions. Turn on persistent approvals for fetch-heavy agents, standardize hook decisions, and separate browser-backed tasks from unattended runs.
