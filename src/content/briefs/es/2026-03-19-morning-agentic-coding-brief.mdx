El mayor cambio de las últimas 24 horas fue estructural: las herramientas de agentes se están moviendo detrás de límites de ejecución más claros en vez de depender de comportamientos locales ad hoc. OpenAI y Google endurecieron cómo se conectan shell, sandbox y agentes delegados, mientras Anthropic volvió a Claude Code más confiable para automatizaciones reales y handoffs entre terminal y editor.

- **Codex está formalizando la ejecución remota alrededor de un exec server**: `0.116.0-alpha.10` y `.11` suman el nuevo stack `exec-server` y eliminan el transporte stdio deprecado en esa ruta. Si corrés Codex en sandboxes hospedados o clientes propios, empezá a apuntar a ejecución tipo JSON-RPC o websocket en vez de asumir stdio local.
- **Codex ya expone los comandos `!` de shell desde el app-server**: OpenAI agregó `thread/shellCommand` al API para que esos atajos corran dentro del sandbox y sigan siendo visibles para el modelo. Si tu cliente todavía maneja escapes de shell por fuera, conviene migrarlos a la superficie del app-server.
- **Claude Code 2.1.79 sí cambia flujos scriptados y con handoff**: corrige el cuelgue de `claude -p` como subprocess, mejora timeouts de fallback no streaming, baja el uso inicial de memoria en unos 18 MB y suma `/remote-control` en VS Code más auth de Console con `--console`. Actualizá si incrustás Claude Code en automatización Python o pasás sesiones del terminal al navegador.
- **Gemini CLI nightly está convirtiendo contexto y navegador en superficies de política**: el último nightly agrega descubrimiento just-in-time de contexto desde `GEMINI.md`, verificación de integridad de extensiones y restricciones de dominios permitidos para el agente de navegador, mientras commits recientes endurecen el sandbox en macOS y la aislación de subagentes. Mové la guía del repo a archivos de memoria por carpeta, definí allowlists de dominios y volvé a probar cualquier flujo multiagente sobre nightly.
