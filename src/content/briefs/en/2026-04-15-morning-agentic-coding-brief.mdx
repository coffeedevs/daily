Today’s useful changes are about making agents more dependable to run, resume, and debug. The pattern is consistent across vendors: keep more behavior in code, make session state survive interruptions, and stop treating agent workflows as one giant prompt.

- **Claude Code makes interrupted sessions cheaper to resume**: `2.1.108` adds `/recap`, moves prompt-cache TTL to 1 hour across API key, Bedrock, Vertex, and Foundry, and exposes built-in slash commands through the Skill tool. Turn on the longer cache path and treat slash commands as reusable agent primitives.
- **Claude Code keeps reducing restart friction**: `2.1.109` improves the extended-thinking indicator, while `2.1.108` narrows `/resume` to the current directory and cuts file-read memory usage. If your team leaves agents running in parallel, standardize resume-heavy workflows instead of restarting threads.
- **Codex is getting friendlier to local extension stacks**: OpenAI landed local marketplace sources, filesystem-aware skill loading, and better tool-search defaults on `main`. Move private plugins and skills closer to the repo and test them as portable workspace assets.
- **Codex added opt-in wire tracing for realtime debugging**: a new trace target logs raw WebSocket events and WebRTC SDP requests without turning on noisy global logs. Add that flag to your repro playbook for stuck voice, browser, or background-agent sessions.
- **Google is pushing agent architecture over prompt ornamentation**: its latest Agent Bake-Off guidance favors decomposed agents, deterministic code around model calls, explicit tool contracts, and fast eval loops. Refactor “do everything” agents into planner/executor/evaluator stages before adding more prompt complexity.
