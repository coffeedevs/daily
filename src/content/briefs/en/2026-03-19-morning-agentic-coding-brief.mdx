The biggest shift in the last 24 hours was structural: agent tools are moving behind clearer execution boundaries instead of ad hoc local behaviors. OpenAI and Google both tightened how shells, sandboxes, and delegated agents are wired, while Anthropic made Claude Code safer to script inside real automation and editor handoff flows.

- **Codex is formalizing remote execution around an exec server**: `0.116.0-alpha.10` and `.11` add the new `exec-server` stack and remove deprecated stdio transport from that path. If you run Codex in hosted sandboxes or custom clients, start targeting JSON-RPC or websocket-style execution instead of local stdio assumptions.
- **Codex landed server-side `!` shell commands on main**: OpenAI added `thread/shellCommand` to the app-server API so shell shortcuts execute inside the sandbox and stay visible to the model. If your client still handles shell escapes out-of-band, plan to route them through the app-server surface.
- **Claude Code 2.1.79 is a real upgrade for scripted and handoff-heavy flows**: it fixes `claude -p` hanging in subprocesses, improves non-streaming fallback timeouts, cuts startup memory by about 18 MB, and adds VS Code `/remote-control` plus Console auth via `--console`. Upgrade now if you embed Claude Code in Python automation or hand sessions from terminal to browser.
- **Gemini CLI nightly is turning context and browser access into policy**: the latest nightly adds just-in-time context discovery from `GEMINI.md`, extension integrity verification, and browser-agent allowed-domain restrictions, while recent commits add stricter macOS sandboxing and more resilient subagent isolation. Move repo guidance into folder-scoped memory files, add browser allowlists, and retest any multi-agent workflow on the nightly branch.
