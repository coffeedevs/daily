El cambio útil de hoy es operativo, no de benchmarks. OpenAI, Anthropic, GitHub y Cursor publicaron ajustes de workflow que hacen más fácil mantener vivos a los agentes, dirigirlos desde otro lugar y repartir trabajo entre tareas concurrentes. Si tu setup todavía asume una sola terminal, una sola rama y una sola sesión en primer plano, hoy eso ya es el cuello de botella.

- **Codex se vuelve más seguro para sesiones locales de larga duración**: OpenAI pasó la consolidación de memoria de Fase 2 a `gpt-5.4` y dejó de descargar threads del app-server apenas se desconecta el último cliente. Reprobá automatizaciones con memoria y tratá la reconexión como continuidad de sesión, no como reinicio en frío.
- **Claude Code suma handoffs reales entre worktrees y guardas de compaction**: la última release agrega `EnterWorktree(path)`, deja que hooks de PreCompact bloqueen compactaciones malas y arranca monitores de plugins en automático. Actualizá hoy hooks y manifests si dependés de worktrees múltiples o supervisión MCP en background.
- **Copilot CLI ya no queda atado a la terminal donde arrancó**: `copilot --remote` ahora permite monitorear y dirigir sesiones vivas desde GitHub web o mobile. Usalo para tests largos, deploys o corridas de agentes que sobreviven al contexto de tu laptop.
- **GitHub mete reparación de PRs con cloud agents más adentro del review loop**: los PRs ahora pueden invocar a Copilot para resolver merge conflicts y pushear fixes desde el entorno cloud de GitHub. Sacá del entorno local la limpieza de conflictos de bajo riesgo y el “fix failing tests”, pero mantené branch protections estrictas.
- **Cursor empuja más fuerte la gestión paralela de agentes**: Cursor 3.1 agrega paneles tileados persistentes y mejor dictado por voz en la ventana de Agents. Si ya dividís trabajo entre agentes, pasá a un workflow de un panel por tarea en vez de saltar pestañas en serie.
